# Security Policy

## Security at ECLYPSA

Security is a core value of the ECLYPSA project.

We appreciate responsible security research and welcome reports of potential vulnerabilities.

---

## Supported Versions

| Version | Supported |
|---------|-----------|
| v0.x | ✅ Yes |
| v1.x | ✅ Yes |

---

## Reporting a Security Issue

If you discover a potential security issue, please do not disclose it publicly before it has been reviewed.

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or logs (if available)
- Suggested fix (optional)

---

## Responsible Disclosure

Please allow reasonable time for the issue to be reviewed and addressed before public disclosure.

We aim to investigate reports as quickly as possible.

---

## Scope

ECLYPSA focuses on:

- Defensive Security
- Authorized Security Assessments
- Security Automation
- Digital Forensics
- Incident Response
- Security Education
- Open Source Research

---

## Thank You

We appreciate everyone who helps improve the security and quality of ECLYPSA.
