# Contributing to ECLYPSA

Thank you for your interest in contributing to ECLYPSA.

We welcome developers, students, security researchers, technical writers, designers, testers, and open-source enthusiasts.

## Our Mission

Build a modern, open-source, AI-extensible platform for authorized defensive security, education, automation, and research.

---

## Before You Start

Please:

- Read the README.
- Review the project roadmap.
- Search existing issues before creating a new one.
- Keep discussions respectful and constructive.

---

## How You Can Contribute

You can contribute by:

- Fixing bugs
- Improving documentation
- Writing tests
- Reviewing code
- Improving performance
- Designing UI/UX
- Creating plugins
- Suggesting new ideas
- Translating documentation

---

## Coding Standards

- Write clean and readable code.
- Keep modules small and maintainable.
- Add comments when necessary.
- Follow the project architecture.
- Write meaningful commit messages.

---

## Pull Requests

Before opening a Pull Request:

- Test your changes.
- Update documentation if required.
- Keep changes focused on a single topic.
- Explain why the change is needed.

---

## Community Values

We value:

- Respect
- Transparency
- Collaboration
- Learning
- Security
- Privacy
- Open Source

Thank you for helping make ECLYPSA better.
